# Palhelm v0.5.0

Palhelm 0.5.0 adds the opt-in Palworld 1.0 Game Data integration, exact-linked live base workers,
operator diagnostics, aggregate activity history, and robust Pal icon variant resolution.

## Upgrade and rollback

Back up the complete `/data` volume before upgrading. This release applies schema migration 009,
which adds `game_data_activity`. The table stores only aggregate FPS/count/activity/link-coverage
samples, prunes data older than 30 days, and never stores actor identity, names, health, guilds,
raw actions, or locations.

Older 0.4.x binaries fail closed after migration 009. To roll back, stop only the panel, restore
the pre-0.5.0 `/data` backup, select the old image tag, and recreate only the `palhelm` service.
Do not restart the Palworld game service as part of a panel rollback.

Before promoting a release candidate, run the read-only contract smoke from a trusted host. Pass
credentials through environment variables so they never appear in shell arguments or output:

```sh
PALHELM_SMOKE_BASE_URL=https://panel.example.test \
PALHELM_SMOKE_ADMIN_PASSWORD='...' \
PALHELM_SMOKE_INTEGRATION_KEY='phk_...' \
node scripts/smoke-release.mjs
```

The smoke logs endpoint names and status only. It does not print response bodies or credentials,
and it does not call any backup, configuration, announcement, save, restore, or server action.

## Optional Game Data capability

Game Data remains opt-in. One shared poller performs the upstream request; panel and Integration
API consumers read its bounded cache. The public `/world/summary` route contains aggregates only,
while `/world/workers` contains only exact save-linked worker identity and owner/base provenance.
Runtime actor IDs, player/platform identifiers, IP addresses, locations, guild/trainer names, and
raw actions are structurally excluded from the Integration API.

The authenticated panel also records aggregate activity diagnostics. The history endpoint returns
at most 500 downsampled rows, and the map ignores stale or truncated live actor snapshots.

## Pal portraits

Boss/Alpha save IDs resolve to base-species art and known cosmetic/tower aliases are normalized.
The dataset roster is the union of canonical Pal IDs and actual installed PNG/WebP files, allowing
operators to add attributed supplemental portraits without changing Palhelm.
