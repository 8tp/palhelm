# Palhelm 0.9.1

A small correctness patch to condenser star ratings.

## Absent rank means never condensed, not unknown

- 0.9.0 treated a Pal whose save carried no `Rank` property as "rank unavailable" and
  showed **Unavailable** in the detail panel. Real save data disproves that reading:
  Unreal's GVAS format omits properties at their default value, and a production 1.0
  save carried 51 pals with an explicit rank of 2–5 and **zero** with an explicit
  rank of 1 across 2,330 pals. The game simply doesn't write the rank until a Pal has
  been condensed.
- The parser now defaults an absent `Rank` to 1, so a never-condensed Pal shows an
  empty four-star row (zero stars) in the detail panel instead of **Unavailable**.
- Compact contexts (explorer cards, box cells, guild Pal cards) still show stars only
  for condensed Pals, so lists stay clean.
- A `null` rank now means exactly one thing: the row was written by a parser from
  before rank decoding shipped. It disappears after the next save parse, and the UI
  still reads **Unavailable** for it rather than pretending to know.

No schema change; no migration. Upgrading from 0.9.0 requires nothing beyond the new
image — ranks correct themselves on the first save parse.
