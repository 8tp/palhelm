# Palhelm v0.8.0

Palhelm 0.8.0 adds bounded server-wide activity analytics and the next live-map interaction
tranche. It does not add a database migration or require a Palworld server restart.

## Server-wide activity

The new authenticated **Activity** page derives rolling 24-hour, 7-day, and 30-day summaries from
the panel's existing observed sessions. It includes bounded concurrency buckets, browser-local peak
hours, first-observed versus returning counts, top-25 player and current-guild rankings, and
unattributed evidence. The contract exposes `trackingSince`, coverage and guild-attribution labels,
and a defensive 100,000-interval truncation flag. It returns no raw sessions and never presents the
results as lifetime Palworld history.

## Map exploration

- Search online players and save-derived guild bases, then explicitly focus a selected result.
- Fit positioned online players or bases covered by the active tile layer.
- Tap or click to pin Palworld display coordinates and create a shareable link.
- Restore shared `x`, `y`, and tile-layer state without encoding player, guild, or actor identity.
- Use touch-sized, wrapping controls and horizontally scrollable layer chips on narrow screens.

Panel login identities are not assumed to match game characters. Palhelm requires an explicit
online-player selection rather than guessing who the current player is.

This release affects only the panel and its authenticated API. It does not change or restart the
Palworld game service.
