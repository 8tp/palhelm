# Palhelm v0.7.0

Palhelm 0.7.0 adds bounded player-activity summaries, semantic event lanes, and route-level
frontend loading. It does not add a database migration or require a Palworld server restart.

## Observed player activity

Player detail now separates total tracked playtime from panel-observed sessions. The activity
card shows the current session plus exact rolling 24-hour, 7-day, and 30-day overlap totals and
session counts. Long sessions are clamped at each window boundary, recent detail is capped at 20
rows, and `trackingSince` makes the available coverage explicit. These figures do not claim to be
lifetime Palworld history.

## Event lanes

The Events screen groups its single newest-500 corpus into Player activity, Operations & audit,
and Health incidents. Lane counts share the same bounded source, while message search, exact-kind
filtering, and 25-row paging remain available.

## Faster navigation

All authenticated pages now load as route-level JavaScript and CSS chunks while authentication,
the navigation rail, and the Shell remain eager. The measured entry JavaScript fell from 678.13 kB
to 367.98 kB minified before gzip, and the previous over-500-kB chunk warning is gone.

This release affects only the panel. It does not change or restart the Palworld game service.
