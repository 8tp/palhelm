# Palhelm v0.6.0

Palhelm 0.6.0 makes save-derived Pal data easier to inspect, improves map interaction, and adds
a viewer-safe diagnostics surface. It does not add a database migration or require a Palworld
server restart.

## Pal data and explorer

- A new authenticated `GET /api/v1/pals` session endpoint provides bounded, keyset-paginated
  search and filters without expanding the public Integration API.
- The new **Pal explorer** searches every parsed Pal by species or owner and filters placement,
  level, owner evidence, and Standard/Alpha/Lucky/Boss specimens.
- Shared party, Palbox, and explorer details show all available numeric work-suitability levels
  with distinct inline SVG symbols. This is bundled, version-pinned species metadata and remains
  labeled separately from individual save observations.

## Map and diagnostics

- Workers are hidden by default, player/base/worker/Palbox markers use distinct SVG shapes, and
  explicit zoom/fit controls make the map easier to operate.
- Wheel and trackpad zoom is contained by the map, preventing the surrounding page from scrolling
  while the pointer is over the map.
- The new **Diagnostics** route summarizes REST/RCON/save health, parser timing and coverage, Game
  Data freshness/link coverage, and backup freshness using existing safe authenticated contracts.

## Release and bot hardening

- Release automation is prepared for tag-only GHCR images with provenance, SBOM, attestations,
  and digest signing. The read-only smoke script now covers the Pal explorer contract too.
- The companion bot aliases opaque save identifiers before AI tool results reach the model,
  sanitizes accidental ID leakage, and uses the compact `AI Generated` footer while preserving
  deterministic source links.

No panel, bot, Docker container, or Palworld game service is changed merely by checking out this
source. Deployment remains an explicit operator action.
